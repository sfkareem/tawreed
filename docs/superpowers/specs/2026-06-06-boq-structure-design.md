# Slab BOQ Output Structure Design

## 1. Overview
This spec defines the exact structural and stylistic requirements for the Excel workbooks generated by the Slab BOQ Slicer. The tool's primary purpose is structural categorization (slicing) rather than cost estimation.

## 2. Sheet Structure

### 2.1 Cover Sheet
*   **Branding:** App branding ("Slab BOQ Slicer") and developer credit ("Developed by kareemsafwat.com").
*   **App Meta:** Minimal application version data.
*   **Project Meta:** "Project Name" and "Date".
*   **LLM Integration:** The LLM will be instructed to extract the "Project Name" from the uploaded BOQ content. If no name is found, the generated cell will be left blank with a bright yellow background fill (`#FFFF00`) to prompt manual entry by the engineer.

### 2.2 Master Sheet
*   **Purpose:** A single, unified flat list containing every single item from the entire project.
*   **Columns (in order):** `Number`, `Package`, `Item Description`, `Unit`, `Quantity`, `Rate`, `Amount`.
*   **Formulas:** The `Amount` column will be pre-filled with an Excel formula (`=Qty*Rate`) ready for future pricing.

### 2.3 Package Sheets
*   **Purpose:** Dedicated sheets for each LLM-assigned category.
*   **Naming Convention:** `Pkg - [Category Name]` (e.g., `Pkg - Masonry`).
*   **Columns (in order):** `Number`, `Item Description`, `Unit`, `Quantity`, `Rate`, `Amount`.
*   **Formulas:** The `Amount` column will be pre-filled with an Excel formula (`=Qty*Rate`) ready for future pricing.

## 3. Styling & Layout (Best Practices)

### 3.1 Dimensions & Layout
*   **Global Orientation:** All sheets must explicitly be set to Right-to-Left (RTL) mode.
*   **Column Widths:** 
    *   `Package`: 20
    *   `Item Description`: 60
    *   All other standard columns (Number, Unit, Qty, Rate, Amount): 15
*   **Row Heights:** Left unmanaged (Auto-fit) to dynamically adapt to text wrapping.

### 3.2 Alignments
*   **Vertical:** All data cells Top-aligned. All Header cells Center-aligned.
*   **Horizontal:** 
    *   `Number`, `Unit`, `Package` Center-aligned.
    *   `Description` Default RTL aligned (Right).
    *   `Quantity`, `Rate`, `Amount` Standard accounting alignment.
*   **Text Wrapping:** Explicitly enabled on `Description` and `Headers`.

### 3.3 Typography & Colors
*   **Font:** Calibri (Size 11 standard).
*   **Headers:** Bold, Light Grey background (`#E7E6E6`), Thin borders.
*   **Standard Text:** Black (`#000000`), White background, Thin borders.
*   **Financial Formats:** Accounting format with two decimals, commas, and zeroes displayed as a dash `-` (`#,##0.00;(#,##0.00);-`).
*   **Missing Data Warnings:** Empty or zero Quantities/Rates get Bold Red text (`#FF0000`) on a Soft Grey background (`#F2F2F2`).

### 3.4 Print Settings (Page Setup)
*   **Fit to Page:** Configured to fit all columns horizontally onto exactly 1 page wide.
*   **Repeat Headers:** The top Header Row (Row 1) must be locked to repeat at the top of every printed page.
