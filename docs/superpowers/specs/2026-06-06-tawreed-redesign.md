# Tawreed Architecture Redesign Spec

- **Date**: 2026-06-06
- **Status**: Approved
- **Topic**: Configuration, Connection Management, Dynamic Column Mapping, and Terminology Audit

---

## 1. System Context & Overview
Tawreed is a desktop application that breaks construction Bills of Quantities (BOQs) into Work Packages. This spec updates the database model, configuration manager, package processing engine, and user interface to conform with professional terminology, remove security risks, and ensure robust multi-sheet mapping.

---

## 2. Component Design & Changes

### A. Database Connection State Management
- **Managed State**: During Tauri initialization, we open `tawreed.db` once, enable Write-Ahead Logging (WAL) mode, and store it in a thread-safe `DbState` structure:
  ```rust
  pub struct DbState {
      pub conn: std::sync::Mutex<rusqlite::Connection>,
  }
  ```
- **Tauri Integration**: Managed via `tauri::Builder::default().manage(...)`.
- **Command Refactoring**: Commands `get_history` and `process_boq` consume `state: tauri::State<'_, DbState>` rather than initializing independent connection handlers.

### B. Configuration Storage
- **Migration**: Configuration details are completely removed from SQLite. The `settings` table is dropped.
- **File-based config**: Configured settings are stored exclusively in `~/.tawreed/config.json`.
- **Handling Defaults**: If `config.json` does not exist on initialization, default settings (MiniMax-M3 base URL and model) are saved.

### C. Dynamic Worksheet Column Mapping
- **The Issue**: Currently, column mappings are resolved once for the first sheet contributing to a package, resulting in scrambled data when processing rows from sheets with different column orderings.
- **The Fix**: The mapping function resolves column indices dynamically for each row's source worksheet inside the extraction loop.

### D. Security & Credential Management
- **Credentials cleanup**: All hardcoded API keys are removed from scripts (`build_test.cjs`) and autogenerated test targets.
- **Git Ignore**: Root `.gitignore` is updated to exclude `.next`, debug caches, local settings, and autogenerated binary files.
- **Public GitHub Push**: Setup and push code to `sfkareem/tawreed` public repository.
